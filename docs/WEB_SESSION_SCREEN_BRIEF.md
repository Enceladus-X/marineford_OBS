# Marineford OBS 통합 웹 세션 전달 문서

이 문서는 기존 웹 페이지에 Marineford OBS 소개, 다운로드, 송출 미리보기, 화면별 설명을 통합하기 위해 다른 웹 작업 세션에 전달하는 단일 브리프입니다.

## 복사해서 전달할 프롬프트

```text

- 단순 다운로드 영역이 아니라, 사용자가 "현장에서 어떤 화면을 누가 어떻게 쓰는지" 이해할 수 있어야 한다.
- 실제 로컬 서버나 매장 데이터를 웹에 올리지 않는다.
```

## 제품 설명 기준

**짧은 설명**

Marineford OBS는 TCG 매장의 방송 테이블에서 사용하는 로컬 송출 보조 도구입니다. 송출컴에서 서버를 실행하고, OBS는 브라우저 소스로 오버레이를 읽으며, 태블릿은 QR로 접속해 경기 진행 상태와 승패 보고를 조작합니다.

**강조할 가치**

- 태블릿 조작으로 송출 화면과 편집 로그가 동시에 갱신된다.
- 듀얼 구간만 편집하기 쉽도록 시작/종료 이벤트가 자동으로 기록된다.
- 사이드 교체중, 저지 호출, 다음 라운드 대기 상태를 OBS 화면에 명확히 표시한다.
- 덱 이름과 이미지는 프리셋으로 관리할 수 있어 반복 세팅 시간을 줄인다.
- 매장 내부 로컬 네트워크에서 동작하므로 웹 미리보기와 실제 운영 데이터는 분리된다.

## 화면별 문서화 내용

### 1. 송출용 컨트롤 패널

- 실제 파일: `control.html`
- 로컬 주소: `http://127.0.0.1:8000/control.html`
- 사용자: 송출 담당자, 운영 스태프
- 목적: OBS 연결, 태블릿 연결, 대회/룰/플레이어/덱 이미지/프리셋/백업을 한 곳에서 관리

**주요 영역**

- 상단 바로가기
  - `송출 화면`, `태블릿 화면`, `로그 패널`
  - `백업`, `복원`, `전체 초기화`
- OBS 화면 연결
  - OBS 브라우저 소스 URL 표시
  - URL 복사, 미리보기 열기
  - 권장 크기: `1920 x 1080`
- 태블릿 연결
  - 태블릿 접속 QR
  - 태블릿에서 열 주소
  - QR 새로고침, 태블릿 패널 열기
- 대회 및 룰 설정
  - 대회명
  - 저장된 룰 프리셋 불러오기
  - BO1/BO3
  - 기본 타이머, 엑스트라 턴 시간
- 플레이어 1 / 플레이어 2
  - 닉네임
  - 덱 이름
  - OBS 상단 닉네임 미리보기
  - 좌측/우측 덱 이미지
  - 이미지 파일 선택, 자르기, 좌우반전
  - 덱 프리셋 저장
- 덱 프리셋 확인 및 수정
  - 덱 이름 검색
  - 저장된 프리셋 목록
  - 왼쪽 적용, 오른쪽 적용, 수정 저장, 삭제
- 수동 점수 조절
  - 좌우 점수 +/-
  - 점수 초기화
- 타이머
  - 시작/일시정지
  - 리셋
  - 현장 진행은 태블릿의 듀얼 시작/저지 호출 흐름을 우선 사용

**웹 페이지에서 보여줄 요약 문구**

송출용 컨트롤 패널은 운영자가 경기 시작 전 세팅을 끝내는 화면입니다. OBS URL, 태블릿 QR, 룰 프리셋, 플레이어 정보, 덱 이미지, 백업/복원을 한 곳에서 관리합니다.

### 2. 태블릿 패널

- 실제 파일: `tablet.html`
- 로컬 주소: `http://송출컴_IP:8000/tablet.html`
- 사용자: 방송 테이블에 앉은 플레이어 또는 현장 진행자
- 목적: 경기 진행 중 손이 많이 가는 조작만 큰 버튼으로 처리

**주요 화면과 상태**

- 경기 전 설정 화면
  - 좌우를 `LEFT`, `RIGHT` 카드로 나눔
  - 각 플레이어가 자신의 영역을 터치해 닉네임과 덱 이름 입력
  - 덱 이름 입력은 상대에게 보이지 않는다는 안내가 있음
  - 양쪽 닉네임과 덱 이름이 입력되어야 `듀얼 시작` 버튼 사용 가능
- 플레이어 설정 모달
  - 닉네임 입력
  - 덱 이름 입력
  - 저장된 덱 프리셋 칩 자동 표시
  - 취소/저장
- 듀얼중 화면
  - 상단 타이머
  - 좌우 플레이어 이름
  - 중앙 대형 세트 스코어
  - 하단 `승패 보고` 버튼
- 승패 보고 모달
  - 실제 닉네임을 사용한 승자 선택
  - 취소
  - 매치 종료
- 사이드 교체중
  - 세트 종료 후 상태
  - 하단 `다음 듀얼 시작` 버튼
- 매치 종료
  - 하단 `다음 라운드 준비` 버튼
  - 누르면 점수, 상태, 닉네임, 덱 이름 초기화
- 저지 호출
  - 상단의 `저지 호출` 버튼
  - 호출 중에는 버튼이 `저지 복귀`로 바뀜
  - 타이머가 멈추고 OBS에는 저지 호출 중 화면 표시
- 고급 모달
  - 전체화면
  - 마지막 보고 취소
  - 새로고침
  - 왼쪽/오른쪽 플레이어 정보 수정
  - 수동 점수 조절
  - 매치 강제 종료

**웹 페이지에서 보여줄 요약 문구**

태블릿 패널은 방송 테이블에서 직접 누르는 진행 화면입니다. 경기 전에는 플레이어가 자신의 닉네임과 덱 이름을 입력하고, 경기 중에는 승패 보고와 저지 호출만 큰 버튼으로 처리합니다.

### 3. OBS 오버레이

- 실제 파일: `overlay.html`
- 로컬 주소: `http://127.0.0.1:8000/overlay.html`
- 사용자: OBS 브라우저 소스
- 목적: 시청자에게 현재 경기 상태, 스코어, 타이머, 플레이어명, 덱 이미지를 보여줌

**표시 요소**

- 좌측/우측 플레이어 이름
- 좌측/우측 덱 이름
- 중앙 타이머
- 현재 상태와 세트 번호
- 세트 스코어
- 좌우 덱 이미지
- 상태 배너

**상태별 화면**

- `dueling`: 일반 듀얼 화면
  - 플레이어명, 덱명, 타이머, 스코어, 덱 이미지 표시
- `sideboarding`: 사이드덱 교체중
  - 중앙 반투명 띠지 형태의 안내 배너
  - 편집에서는 잘라낼 수 있지만 라이브 시청자는 대기 상태를 이해할 수 있음
- `judge_call`: 저지 호출 중
  - 중앙 반투명 띠지 형태의 안내 배너
  - 타이머는 멈춘 상태
- `round_waiting`: 다음 라운드 대기중
  - 전체 암전형 대기 화면
- `finished`: 매치 종료
  - 전체 암전형 종료 화면
  - 최종 스코어 표시

**웹 페이지에서 보여줄 요약 문구**

OBS 오버레이는 실제 방송에 얹히는 화면입니다. 듀얼중에는 점수와 타이머를 작게 유지하고, 사이드 교체나 저지 호출처럼 흐름이 끊기는 순간에는 중앙 배너로 상태를 명확히 표시합니다.

### 4. 편집 로그 패널

- 실제 파일: `editor.html`
- 로컬 주소: `http://127.0.0.1:8000/editor.html`
- 사용자: 송출자, 편집자
- 목적: 태블릿 조작 기록을 바탕으로 편집점과 유튜브 업로드 보조 자료 생성

**주요 영역**

- 상단 바로가기
  - 컨트롤
  - 송출 화면
  - 태블릿
- 내보내기
  - `cuts.csv`
  - `chapters.txt`
  - `ffmpeg.txt`
- 녹화 기준 찍기
  - 편집 기준 시각을 로그에 기록
- 요약 카드
  - 현재 상태
  - 현재 세트
  - 스코어
  - 듀얼 구간 수
- 듀얼 구간 테이블
  - 세트
  - 시작
  - 종료
  - 승자
  - 스코어
  - 길이
- 이벤트 로그
  - 듀얼 시작
  - 세트 종료
  - 매치 종료
  - 다음 라운드 준비
  - 저지 호출 시작/종료
  - 타이머 조작
  - 수동 점수 조절

**웹 페이지에서 보여줄 요약 문구**

편집 로그 패널은 방송 후 영상을 빠르게 자르기 위한 보조 화면입니다. 태블릿에서 누른 듀얼 시작과 승패 보고가 로그로 남고, CSV, 챕터 초안, ffmpeg 명령 초안을 내려받을 수 있습니다.

방송 테이블에서 직접 누르는 진행 화면입니다. 플레이어 설정, 듀얼 시작, 승패 보고, 저지 호출을 큰 버튼 중심으로 처리합니다.

**OBS 오버레이**

시청자가 보는 방송용 화면입니다. 듀얼중에는 스코어와 타이머를 보여주고, 사이드 교체나 저지 호출 중에는 중앙 배너로 현재 상태를 표시합니다.

**편집 로그 패널**

방송 후 편집 시간을 줄이는 보조 화면입니다. 듀얼 구간, 챕터 초안, ffmpeg 명령 초안을 이벤트 로그에서 자동으로 정리합니다.

## 임베드 구현 상세

### 권장 코드

```html
<div id="marineford-preview"></div>
<script src="https://enceladus-x.github.io/marineford_OBS/preview/embed.js"></script>
```

### 옵션 예시

```html
<div id="marineford-preview"></div>
<script
  src="https://enceladus-x.github.io/marineford_OBS/preview/embed.js"
  data-target="#marineford-preview"
  data-default-demo="duel"
  data-title="Marineford OBS"
  data-description="TCG 매장 방송 송출용 로컬 운영 도구입니다."
  data-show-steps="true">
</script>
```

### 직접 iframe을 써야 할 때

```html
<iframe
  src="https://enceladus-x.github.io/marineford_OBS/preview/overlay.html?demo=duel&v=latest"
  title="Marineford OBS 송출 화면 미리보기"
  style="width:100%;aspect-ratio:16/9;border:0;border-radius:8px;background:#050506;"
  loading="lazy">
</iframe>
```

직접 iframe 방식은 캐시 갱신이 약하므로 가능하면 `embed.js`를 사용합니다.

## 공개하면 안 되는 것

- `state.json`
- `events.jsonl`
- `deck_presets.json`
- `rule_presets.json`
- `exports/`
- 현장 업로드 이미지
- 백업 JSON
- 실제 매장/player 데이터

웹 페이지에는 샘플 데이터나 정적 미리보기만 사용합니다.

## 웹 QA 체크리스트

- Marineford OBS 미리보기 위젯이 로드되는가?
- `듀얼중`, `사이드 교체중`, `저지 호출`, `다음 라운드 대기` 버튼이 동작하는가?
- iframe이 데스크톱/모바일 모두 16:9를 유지하는가?

- 실제 운영 데이터 파일이 웹 빌드에 포함되지 않았는가?
- CSP가 있다면 `script-src`와 `frame-src`에서 `https://enceladus-x.github.io`가 허용되는가?
